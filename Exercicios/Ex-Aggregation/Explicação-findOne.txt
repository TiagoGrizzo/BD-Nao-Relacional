O findOne é o método utilizado quando você quer buscar apenas um único documento dentro de uma coleção.Diferente do find(), que retorna um "cursor" (uma lista de resultados que você precisa percorrer), o findOne() entrega o objeto pronto diretamente.Aqui estão os pontos principais para a sua prova:1. Como ele funcionaSe houver mais de um documento que atenda ao critério da busca, o findOne retornará apenas o primeiro que ele encontrar fisicamente no disco.JavaScript// Busca o primeiro prato que encontrar com o nome "Pizza"
db.menu.findOne({ dish: "Pizza" });
2. Diferenças fundamentais (find vs findOne)Característicadb.collection.find()db.collection.findOne()RetornoUm Cursor (múltiplos documentos)Um Objeto JSON únicoSe não achar nadaRetorna um cursor vazioRetorna nullUso comumListagens, relatórios, filtros amplosBuscar por _id ou usuário específico3. Exemplo Prático com ProjeçãoVocê também pode usar o findOne para pegar apenas um campo específico de um documento. Por exemplo, se você quiser apenas o e-mail do aluno "Nome 500":JavaScriptdb.alunos.findOne(
  { nome: "Nome 500" }, // Filtro
  { email: 1, _id: 0 }  // Projeção (mostra email, esconde ID)
);
4. Por que usar o findOne?Performance: O banco para de procurar assim que encontra a primeira ocorrência. Ele não precisa varrer o resto da coleção.Praticidade no Código: Em linguagens de programação (como Node.js ou Python), você não precisa de um loop for ou .toArray() para ler o resultado; o dado já vem "limpo" como um objeto.Dica de Prova:Se o professor perguntar: "O que acontece se eu usar findOne({}) sem nenhum filtro?"Resposta: Ele retornará o primeiríssimo documento que foi inserido na coleção (ou o primeiro disponível no índice padrão).Ficou claro? O findOne é basicamente o comando de "busca cirúrgica" do MongoDB!
